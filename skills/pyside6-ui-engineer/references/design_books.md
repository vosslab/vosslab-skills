# Design books

Use this reference when a PySide6 interface needs stronger visual judgment, better component logic, cleaner typography, or more systematic UI decisions.

## How to use this list

- Do not treat books as style law.
- Use them to sharpen judgment about hierarchy, spacing, typography, grids, interaction, and design systems.
- Prefer practical books first when the task is active implementation.
- Prefer classic books when the task needs deeper design reasoning.

## Best immediate references

- `Refactoring UI` by Adam Wathan and Steve Schoger
  Use for practical fixes to spacing, visual hierarchy, color usage, layout density, and component polish.
  Local file available: [`references/Refactoring_UI.txt`](Refactoring_UI.txt)
- `Practical UI` by Adham Dannaway
  Use for logic-driven rules that map well to implementation work and design critique.
  Local file available: [`references/Practical_UI.txt`](Practical_UI.txt)
- `How to Design Better UI Components` by Adrian Kuleszo
  Use for concrete component-level improvements and visual examples.
- `UI Design Principles` by Michael Filipiuk
  Use for broader modern UI guidance across typography, color, and layout.

## Typography references

- `Better Web Typography for a Better Web` by Matej Latin
  Use for readable type pairing, paragraph structure, rhythm, and hierarchy.
- `Thinking with Type` by Ellen Lupton
  Use for stronger fundamentals in type composition and emphasis.
- `The Visual History of Type` by Paul McNeil
  Use when deeper type knowledge helps explain why a visual choice feels right or wrong.

## Layout and composition references

- `Grid Systems in Graphic Design` by Josef Muller-Brockmann
  Use for alignment discipline, spacing logic, and systematic composition.
- `The Non-Designer's Design Book` by Robin Williams
  Use for contrast, repetition, alignment, and proximity when a UI lacks clarity.

## Systems and scale references

- `UI Design Systems Mastery` by Marina Budarina
  Use for design-system structure, documentation, and component consistency.
- `Design Systems` by Alla Kholmatova
  Use for cross-team design-system thinking and sustainable component language.
- `Laying the Foundations` by Andrew Couldwell
  Use for documenting and maintaining design systems over time.
- `Atomic Design` by Brad Frost
  Use for thinking about component granularity and reusable interface building blocks.

## Interaction and behavior references

- `About Face` by Alan Cooper, Robert Reimann, David Cronin, and Christopher Noessel
  Use for interaction design framing, task flows, and goal-directed behavior.
  Local file available: [`references/About_Face.txt`](About_Face.txt)
- `UI is Communication` by Everett N McKay
  Use when wording, control meaning, and screen communication are weak.
- `Universal Principles of Design` by William Lidwell, Kritina Holden, and Jill Butler
  Use for cross-disciplinary design and psychology principles that explain user behavior.

## Focused niches

- `The Design Manual` by Adrian Kuleszo
  Strong reference for mobile-first visual and product design thinking.
- `The Ultimate Guide to Web Design` by Adrian Kuleszo
  Useful when a desktop UI borrows patterns from responsive web design.
- `Master UI Design Elements` by Denislav Jeliazkov
  Good quick-start reference for practical UI fundamentals.
- `Landing Page Hot Tips` by Rob Hope
  Useful for conversion-oriented messaging and visual sequencing patterns.
- `100 UI/UX Tips & Tricks` by Victor Ponamariov
  Good short-form prompts for critique passes and visual cleanup.

## Practical mapping to PySide6 work

- If the interface feels cluttered or visually uneven, start with the local trio: `Refactoring UI`, `Practical UI`, and `About Face`.
- If text hierarchy feels weak, start with `Better Web Typography for a Better Web` and `Thinking with Type`.
- If a component library or reusable widget set is emerging, start with `Atomic Design`, `Design Systems`, and `UI Design Systems Mastery`.
- If layouts feel inconsistent, start with `Grid Systems in Graphic Design`.
- If the product flow is confusing, start with `About Face`, then `UI is Communication`.
